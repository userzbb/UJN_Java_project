## 2. BLE 频谱感知 — 通信原理

### 2.1 BLE 协议基础

BLE（Bluetooth Low Energy，蓝牙低功耗）工作在 2.4 GHz ISM 频段（2400–2483.5 MHz），共有 40 个信道，每个信道带宽 2 MHz。

```
BLE 40 信道划分：
┌──────────────────┬──────────────────────────────────┐
│  信道 37/38/39    │  信道 0–36                       │
│  广播信道          │  数据信道                         │
│  (Advertising)    │  (Data)                          │
│                   │                                  │
│  37: 2402 MHz     │  用于连接建立后的双向数据通信        │
│  38: 2426 MHz     │                                  │
│  39: 2480 MHz     │                                  │
└──────────────────┴──────────────────────────────────┘

频谱位置：
2402 MHz ← 37 ── 38 ── 39 → 2480 MHz
          ↑      ↑      ↑
       低端     中段    高端
```

**为什么广播信道选 37/38/39？**
- 刻意避开 WiFi 最常用的信道 1/6/11（2412/2437/2462 MHz），减少同频干扰
- 分散在 80 MHz 带宽两端和中段，频率分集

### 2.2 广播包结构（ADV PDU）

```
┌──────────┬──────────┬────────────┬─────────────────────┬──────────┐
│ Preamble │ Access   │ PDU Header │ PDU Payload (AD)     │ CRC      │
│ (1 字节)  │ Address  │ (2 字节)    │ (0-31 字节)          │ (3 字节)  │
│          │ (4 字节)  │            │                      │          │
└──────────┴──────────┴────────────┴─────────────────────┴──────────┘

PDU Payload = AdvA (6 字节) + AdvData (0-31 字节)
AdvA: 广播者 MAC 地址
AdvData: AD Structure 序列
```

**AD Structure 格式：**

```
┌──────────┬─────────┬─────────────┐
│ Length   │ AD Type │ AD Data     │
│ (1 字节)  │ (1 字节) │ (Length-1 B)│
└──────────┴─────────┴─────────────┘

常见 AD Type：
  · 0x01 — Flags（发现模式、BR/EDR 支持）
  · 0x09 — Complete Local Name（设备名如 "AirPods"）
  · 0x0A — Tx Power Level（发射功率，dBm）
  · 0xFF — Manufacturer Specific Data（厂商自定义，Apple/Google 常用）
  · 0x16 — Service Data（16-bit UUID，如 Exposure Notification）
```

### 2.3 RSSI 与距离估计

**RSSI（Received Signal Strength Indicator）** 是接收端测量到的信号强度，单位 dBm。

```
自由空间路径损耗模型（Friis formula）：

    RSSI = TxPower - 10·n·log₁₀(d) - L₀

    其中: TxPower = 发射功率（通常 0 dBm）
          n       = 路径损耗指数（室内 ~2.5–3.5）
          d       = 距离（米）
          L₀      = 参考距离损耗（≈ 40 dB @ 1m, 2.4GHz）
```

**实际工程中：** RSSI 不能精确换算距离——多径反射、天线方向性、人体遮挡都会导致 ±10 dB 的波动。项目里使用 RSSI 做**粗粒度距离分级**（近/中/远），不做精确定位。

### 2.4 信道占用率计算

```
信道占用率 = (该信道收到的广播包数 / 总广播包数) × 100%

统计窗口：30 秒滑动窗口

前端展示：3 列柱状图（37/38/39），高度 = 占用率百分比
```

## 3. WiFi CSI 人体感知 — 通信原理

### 3.1 从 RSSI 到 CSI

| | RSSI | CSI |
|------|------|-----|
| **粒度** | 整个帧的接收功率（一个数） | 每个子载波的幅度+相位（矩阵） |
| **维度** | 1 维 | N_tx × N_rx × N_subcarriers 维 |
| **信息量** | 极低，只知道"信号有多强" | 极高，能感知多径传播的细微变化 |
| **灵敏度** | 人走过可能变 2-3 dB | 子载波能检测到毫米级的人体位移 |

```
RSSI: "信号强度 -45 dBm"
CSI:  [H₁₁, H₁₂, ..., H₁₂₅₆]    ← 每个子载波一个复数，256 个独立测量
       ↑     ↑           ↑
      幅度+相位 @ 子载波 1, 2, ..., 256

CSI 矩阵维度（Intel AX200, 2×2 MIMO, 80MHz）:
  N_tx = 2
  N_rx = 2
  N_subcarriers = 256 (HT/VHT) / 1992 (HE with AX200)
```

### 3.2 OFDM 与子载波

CSI 之所以有"子载波"维度，是因为 WiFi 使用 OFDM（正交频分复用）。

```
OFDM 发射端：
比特流 → 串并转换 → QAM调制 → IFFT → 加CP → DAC → RF
                              ↑
                        每个子载波独立调制

子载波编号（20MHz, 64 子载波为例）：
          -32  ...  -1  0  1  ...  31
非活跃  ┌─────┬─────┬─────┬─────┬──────┐
        │ DC  │ Pil │Dat │ Dat │ ...  │
        │ 0   │ -21 │ -2 │  1  │      │
        └─────┴─────┴─────┴─────┴──────┘

802.11ax (WiFi 6) 80MHz: 996 个活跃数据子载波
```

接收端解调出每个子载波的复数信道响应 H(k)，CSI 矩阵就是所有 H(k) 的集合。

### 3.3 CSI 幅度与相位

```
对于每个子载波 k：
    H(k) = |H(k)| · e^(j∠H(k))
           ↑            ↑
        幅度          相位

幅度 |H(k)|:
  · 反映该频率的信号衰减
  · 频率选择性衰落 → 不同子载波幅度不同
  · 人移动会改变多径 → 某些子载波幅度波动

相位 ∠H(k):
  · 反映信号传播时延
  · 微小位移（毫米级）引起显著相位变化
  · 需要相位解缠去除 2π 跳变

CSI 幅度热力图示例（y=子载波索引, x=时间, 颜色=幅度）:

时间 →
子 ↑   ░░████████████████░░░░░     ← 人在时某些子载波幅度下降
载 │   ░░░░░░██████░░░░░░░░░░
波 │   ░░░░░░░░░░████████░░░░░
    │   ░░░░░░░░░░░░░░████░░░░
```

### 3.4 人体运动检测原理

**核心思想：CSI 幅度方差 ∝ 环境变化（人的运动）。**

```
算法步骤：

1. 原始 CSI 采集: H[t] = [h₁, h₂, ..., h₂₅₆], t = 帧序号

2. 幅度提取: A[t] = |H[t]|, 形状 (256,) 的实数向量

3. Hampel 异常值滤波:
   对每个子载波 k:
     median_k = median(A[t-W:t+W])  {滑动窗口内中位数}
     MAD_k    = median(|A[t-W:t+W] - median_k|)
     if |A[t, k] - median_k| > 3·MAD_k → 替换为 median_k

4. 子载波方差:
    var[t] = Var(A[t])  {256 个子载波幅度的方差}

5. 运动检测:
    motion_score[t] = var[t] / var_baseline  {归一化到基准}
    if motion_score[t] > threshold (默认 2.0) → motion_detected = true
```

**为什么方差有效？**
- 无人时：CSI 幅度稳定（仅热噪声），方差低
- 有人走动时：人体反射产生新的多径，某些子载波幅度波动剧烈，方差升高

### 3.5 影响 CSI 灵敏度的因素

| 因素 | 影响 |
|------|------|
| **WiFi 频段** | 5GHz 比 2.4GHz 波长更短 → 对微小位移更敏感 |
| **带宽** | 80MHz 比 20MHz 有更多子载波 → 频率分集更好 |
| **天线数** | 2×2 MIMO 比 1×1 提供空间分集 |
| **发包率** | 发包越密时间分辨率越高，但功耗越大 |
| **环境** | 多径丰富的室内效果最好（反射多变化大） |
